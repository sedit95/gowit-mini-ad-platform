# Planning Workflow

When a new task is initiated, the Planning Agent must:
1. Read AI_CONTEXT.md to understand core boundaries and risks.
2. Check README.md and the docs/ directory for existing context.
3. Identify the exact scope of the request and verify it does not violate scope limitations.
4. Ask the human developer for clarification before making unapproved architectural decisions or implementations.
5. Emphasize simple, explainable architecture and avoid overengineering.
