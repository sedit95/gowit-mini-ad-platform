# Backend (Go + PostgreSQL) Skill Instructions

When working on the backend:
- Use standard Go idioms and standard library features where possible.
- Use raw PostgreSQL queries or a lightweight tool; avoid massive ORMs if not needed.
- Maintain a simple layered structure (e.g., HTTP handlers, service/domain, DB repository).
- Include strict validation and explicit error handling.
- Do not introduce unnecessary abstractions or generic boilerplate.
